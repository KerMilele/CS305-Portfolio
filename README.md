# CS305-Portfolio

1.	Briefly summarize your client, Artemis Financial, and its software requirements. Who was the client? What issue did the company want you to address?

Artemis Financial is a company that provides financial services. They wanted to improve the security of their web application to better protect customer data. The main issue was making sure their system was secure when sending and receiving information, especially by adding encryption and checking for vulnerabilities.

2. What did you do well when you found your client’s software security vulnerabilities? Why is it important to code securely? What value does software security add to a company’s overall well-being?

I did well in identifying possible vulnerabilities in the code, like weak encryption and outdated dependencies. I also followed a clear process to review the system step by step.
Coding securely is important because it protects sensitive data like personal and financial information. It also helps prevent attacks and builds trust with customers. Strong security helps a company avoid data breaches, financial loss, and damage to its reputation.

3. Which part of the vulnerability assessment was challenging or helpful to you?

The most challenging part was understanding all the different types of vulnerabilities and how they could affect the system.
The most helpful part was using tools and following the vulnerability assessment process, which made it easier to find problems.

4.  How did you increase layers of security? In the future, what would you use to assess vulnerabilities and decide which mitigation techniques to use?
I increased security by adding encryption (SSL), checking dependencies for known vulnerabilities, and improving how data is handled.
In the future, I would use tools like OWASP Dependency-Check and follow secure coding practices to assess vulnerabilities and choose the best solutions.

5. How did you make certain the code and software application were functional and secure? After refactoring the code, how did you check to see whether you introduced new vulnerabilities?

I tested the application after making changes to make sure it still worked correctly. I also checked for errors and reviewed the code again to make sure no new vulnerabilities were added.

6. What resources, tools, or coding practices did you use that might be helpful in future assignments or tasks? 
I used tools like OWASP Dependency-Check, Maven, and secure coding practices. These tools helped me identify vulnerabilities and fix them. I also followed best practices like input validation and updating dependencies.

7.  Employers sometimes ask for examples of work that you have successfully completed to show your skills, knowledge, and experience. What might you show future employers from this assignment?

I would show this assignment as an example of my ability to identify security risks and improve software security. It shows that I understand secure coding, vulnerability assessment, and how to protect real-world applications.
